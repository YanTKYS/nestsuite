# NestSuite 利用ガイド

NestSuite は、NoteNest / ChatNest / IdeaNest / TempNest の 4 つの Workspace を 1 つのシェル上で並行利用できる統合インターフェースです。

---

## 起動方法

### 起動（v1.21.0）

```
NestSuite.exe
NestSuite.exe sample.notenest
NestSuite.exe sample.chatnest
NestSuite.exe sample.ideanest
```

- 引数なしで起動すると、常駐する TempNest タブがアクティブな状態で始まります（前回セッションがある場合はそちらを復元します）
- ファイルパスを指定すると、拡張子に応じて対応する Workspace のタブを自動的に開きます
- 未対応拡張子や読込失敗時はエラーを表示し、TempNest タブへフォールバックします

### `--classic-notenest`（v1.19.3 で削除済み）

- v1.19.3 で `--classic-notenest` 起動ルートを削除しました。指定しても NestSuite が通常起動します
- 退避が必要な場合は v1.19.2 以前のリリースを利用してください

### `--nestsuite`（互換）

```
NestSuite.exe --nestsuite
NestSuite.exe --nestsuite sample.notenest
NestSuite.exe --nestsuite sample.chatnest
NestSuite.exe --nestsuite sample.ideanest
```

- `--nestsuite` は v1.6.1 互換として維持されています。既定と同じ動作になります
- 既存のスクリプトやショートカットで `--nestsuite` を使っている場合は、引き続きそのまま動作します

## 4 Workspace とファイル形式

| Workspace | 内容 |
|-----------|------|
| NoteNest | ノート・タスク・マーカーを JSON で管理するプロジェクトファイル（スキーマ 1.4.2） |
| ChatNest | チャット形式のメモ（JSON） |
| IdeaNest | アイデアカード集（JSON） |
| TempNest | 起動中常駐する一時メモ。ファイル保存の対象外 |

NoteNest / ChatNest / IdeaNest は、新規保存時の標準拡張子が `.nestsuite` に統一されています（1 タブ = 1 ファイルのまま、ファイル内容の種別で対応する Workspace を判定します）。旧来の `.notenest` / `.chatnest` / `.ideanest` も引き続き開けます。各 Workspace の保存内容（スキーマ）自体は互いに独立しています。共通プロジェクトファイル形式（複数 Workspace を 1 ファイルにまとめる形式）は現時点では未導入です。

---

## タブの考え方

NestSuite のタブは**ツール単位ではなく、ファイル／作業単位**で作成されます。

```
例：[業務改善.notenest] [会議メモ.chatnest] [アイデア整理.ideanest] [B.notenest]
```

- 同じツールのタブを複数同時に開けます（例：NoteNest を 2 タブ）
- タブごとに独立した ViewModel を持ち、内容が他のタブに混ざることはありません

## 基本操作

### 新規作成

ファイルメニュー → 新規 → ツール別サブメニューから、作成したいツールのタブを選べます（v1.10.1）。

| 操作 | 結果 |
|------|------|
| ファイル → 新規 → 新規 NoteNest | 新規 NoteNest タブを作成 |
| ファイル → 新規 → 新規 ChatNest | 新規 ChatNest タブを作成 |
| ファイル → 新規 → 新規 IdeaNest | 新規 IdeaNest タブを作成 |

サイドバーのツールボタンをクリックしても同じ種別のタブを作成またはフォーカスできます。

### ファイルを開く

ファイルメニュー → 開く から、対応する形式すべてを扱える統合ダイアログが開きます（v1.10.1）。

| 手順 | 内容 |
|------|------|
| ファイル → 開く | `.nestsuite / .notenest / .chatnest / .ideanest` を選択できるダイアログを表示 |
| ファイルを選択 | 拡張子または内容の種別に応じて対応する Workspace のタブを自動作成 |
| 未対応拡張子を選択 | エラーメッセージを表示し、タブは作成しない |

- 選択中タブの種別に関わらず、任意のファイル形式を開けます
- 同じファイルが既に開かれている場合は、新しいタブを作成せず既存タブをアクティブにします

### ファイルを開けないときの案内

ファイルを開けない場合は、ダイアログに表示される形式・場所・権限などの案内を確認してください。
NestSuite で開ける主な形式は `.nestsuite` / `.notenest` / `.ideanest` / `.chatnest` です。
起動時や外部（ファイル関連付け・二重起動）からファイルを開こうとして失敗した場合でも、
NestSuite 自体は起動しています。別のファイルを開くか、新しいタブで作業を開始できます。

複数ファイルをまとめて開いたとき、一部のファイルを開けない場合は、ダイアログに件数・ファイル名・
理由が表示されます（v2.16.15）。成功したファイルは通常どおり開かれます。

前回開いていたファイルが見つからない場合、復元失敗通知の画面で、次回からそのファイルの再試行を
止めるか確認されます（v2.16.21）。ファイルを移動・削除済みの場合は再試行を止められます。外部ドライブや
ネットワークドライブが一時的に未接続の場合は、再試行を続けてください。

#### 復元できないファイルが毎回表示される場合

前回開いていたファイルの復元に失敗する理由には、ファイルが見つからない場合のほか、ファイル形式を
判定できない、アクセス権がない、新しい形式のファイルである、などがあります。

ファイルが見つからない場合は、復元失敗通知の画面で「次回から再試行しない」を選ぶことで、次回以降の
復元対象から外せます。

一方、ファイル形式を判定できない場合やアクセスできない場合などは、現時点では直接「次回から再試行
しない」の対象にはなりません。破損が疑われるファイルは、単体で開き直すと `.bak` からの復元案内が
表示される場合があります。アクセス権やネットワークドライブの問題が疑われる場合は、アクセスできる
状態にしてから再起動してください。

今後も復元する必要がないファイルであれば、必要な内容が残っていないか確認したうえで、該当ファイルを
移動または削除してください。次回起動時に「ファイルが見つからない」状態として扱われれば、復元失敗
通知の画面で「次回から再試行しない」を選べます。

### 最近使ったファイル

ファイルメニュー → 最近使ったファイル から、直近 10 件のファイルにすばやくアクセスできます（v1.14.0）。

- NoteNest / ChatNest / IdeaNest の 3 ツール横断で記録します
- リストは開いた順に先頭へ追加されます（重複は自動排除・先頭移動）
- 記録されたファイルが見つからない場合はエラーを表示し、履歴から自動削除します
- アプリ終了後も次回起動時に引き継がれます（`%APPDATA%\NoteNest\nestsuite-recent-files.json`）

### 保存

| 操作 | 結果 |
|------|------|
| ファイル → 上書き保存 | 選択中タブのみ保存 |
| ファイル → 名前を付けて保存 | 選択中タブを別名で保存 |

- 他のタブの状態には影響しません
- 名前を付けて保存で、別タブで既に開かれているファイルパスを指定した場合はエラーを表示し保存を中止します

### 自動保存と `.bak`

自動保存では通常の保存ファイルを更新しますが、`.bak` バックアップは手動保存時の復元候補として残します。

#### 無題タブの下書き

無題でまだ保存先を持たない NoteNest / IdeaNest / ChatNest タブは、30 秒間隔の自動保存 tick で `%APPDATA%\NoteNest\drafts\` に下書き保存されます。下書きには Workspace と同等の内容が含まれるため、取り扱いには通常の保存ファイルと同じ注意が必要です。正常に保存・タブ閉鎖・アプリ終了できた場合、対応する active 下書きは削除されます。

異常終了後に下書きが残っている場合、次回起動時に件数を通知して復元するか確認します。「はい」を選ぶと無題タブとして復元し、復元後は通常の無題タブと同じように名前を付けて保存できます。「いいえ」を選ぶと下書きを破棄します。「キャンセル」または Esc / ✕ では今回は何もせず保持し、次回起動時にもう一度確認します。

ChatNest は確定済みメッセージに加え、`draft-*.state.json` sidecar から未送信入力、選択中 Speaker、編集中テキストも復元対象にします。ただし、IdeaNest のカード作成・編集ダイアログ内でまだ確定していない入力や、NoteNest のリネーム等のダイアログ内で未確定の入力は対象外です。読めない下書きは削除せず、drafts フォルダー内の `.corrupt-*` ファイルへ退避します。


### `.bak` からの復元

`.bak` は最後に手動保存した時点の内容の復元候補です（自動保存では更新されません）。
ファイルが開けない・破損している可能性があると感じたときは、次の手順で手動復元できます。

1. NestSuite で対象ファイルを閉じます
2. 元のファイルを削除せず、別名（例: `project.notenest.broken`）で退避します
3. `.bak` ファイルをコピーします
4. コピーしたファイル名から `.bak` を外し、元の拡張子に戻します
5. NestSuite で開けるか確認します

詳しい手順と注意点は、ヘルプメニューの「バックアップ復元ガイド」からも確認できます。
NestSuite が `.bak` を自動的に復元・コピーすることはありません。

### 操作完了の表示

保存・すべて保存・コピー・Markdown エクスポート・TempNest のクリアなど主要操作では、
画面下部のステータスバーに「保存しました」「コピーしました」などの短い完了メッセージが
1〜2 秒ほど表示されてから自動的に消えます。失敗時は従来どおりエラーダイアログを表示します。

### 無効なメニュー・ボタンのツールチップ

押せないメニューやボタンには、理由や次に必要な操作を示すツールチップが表示される場合があります。
保存・名前を付けて保存・すべて保存・タブを閉じる・ピン留め／解除など主要な操作が対象です。

### タブを閉じる

- タブの「×」ボタンをクリックします
- 未保存の変更がある場合、確認ダイアログを表示します（タブ名が明示されます）
- キャンセルするとタブは残ります
- 最後の通常タブを閉じると、常駐する TempNest タブがアクティブになります（TempNest は閉じられない固定タブのため常に存在します）

### 終了時の未保存確認

複数の未保存タブがある状態で NestSuite を終了しようとすると、最初に件数の要約が表示されます。
その後、従来どおり各タブごとに保存するか確認します。要約はまとめて保存・破棄するものではなく、
これから何件確認が続くかを事前に把握できるようにするものです。未保存タブが 0 件・1 件の場合は
要約を表示せず、従来どおりの確認のみになります。

### タブをピン留めする

よく使う通常タブは、タブを右クリックして「ピン留め」を選ぶと Temp タブの直後に固定できます。
ピン留め済みタブはタブ名の先頭に「📌」が付き、右クリックメニューの「ピン留めを解除」から通常タブ領域へ戻せます。
Temp タブは固定タブのため、ピン留め操作の対象外です。

## タブのツールチップ

タブにマウスを合わせると、以下の情報が表示されます：

```
種類: NoteNest
ファイル: C:\work\業務改善.notenest
状態: 保存済み
```

未保存タブの場合：
```
種類: ChatNest
ファイル: 未保存（無題）
状態: 保存済み
```

## Shell 横断検索

`Ctrl+Shift+F` で開く横断検索は、**現在開いているタブのみ**が対象です。閉じているファイルや最近使ったファイル履歴は検索対象に含まれません。
