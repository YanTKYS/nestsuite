# NestSuite 既知の制約（v2.18.13）

NestSuite の現時点での到達点と、今後の整理対象となっている制約を記録します。

---

## 現在の到達点

- NoteNest / IdeaNest / ChatNest / TempNest の 4 Workspace を 1 つのシェル上でタブ管理できます
- 各 Workspace は複数ファイルを別タブとして並行利用できます
- タブはファイル／作業単位で独立しており、内容が他のタブに混ざることはありません
- 同じファイルの二重オープンを検出し既存タブへ移動します
- 名前を付けて保存で他タブが開くファイルへの上書きを防ぎます
- 終了時に保存済みファイルタブを記録し、次回起動時に復元します（v1.15.0。引数付き起動でも復元を試みます、v1.18.2）
- 「開く...」ダイアログで複数ファイルを同時選択してまとめて開けます（v1.16.0）
- タブのドラッグによる並び替えができます（v1.17.0）
- NoteNest / IdeaNest / ChatNest のタブを別ウィンドウ（`DetachedWorkspaceWindow`）に切り出して表示できます（v2.9.0〜）
- 新規保存 / 名前を付けて保存の標準拡張子は `.nestsuite` です（v2.14.1〜）。旧 `.notenest` / `.chatnest` / `.ideanest` も互換読み込みできます

---

## 制約一覧

### 起動

| 制約 | 状況 |
|------|------|
| ファイル関連付けの自動設定は未対応 | `.nestsuite` / `.notenest` / `.chatnest` / `.ideanest` の関連付けは、インストーラーによる自動登録ではなく、ヘルプメニューからの手動登録が必要 |

### タブ管理

| 制約 | 状況 |
|------|------|
| タブ復元は保存済みファイルのみ対応（v1.15.0） | 未保存タブ・カーソル位置・ペイン状態は復元されない |
| タブ並び替えはドラッグのみ対応（v1.17.0） | ドラッグ以外の並び替え（キーボード操作等）は未対応 |

### Workspace 別ウィンドウ表示（DetachedWorkspaceWindow）

| 制約 | 状況 |
|------|------|
| TempNest は別ウィンドウ表示に非対応 | TempNest は閉じられない固定タブであるため、分離操作の対象外とする |
| 別ウィンドウ状態はセッションに保存されない | アプリ再起動時に前回の別ウィンドウ配置は復元されない。すべてのタブは Shell に統合された状態で起動する |
| 別ウィンドウは Shell ウィンドウより手前に固定される | Windows の Owner ウィンドウ機構（`Window.Owner`）により、別ウィンドウを Shell の背後に移動することはできない。Shell の最小化・復元連動のためにこの制約を維持している（v2.9.6） |
| 複数ウィンドウのレイアウトは保存されない | 別ウィンドウの位置・サイズはセッション保存の対象外 |
| 常に最前面（Always on Top）は非対応 | 別ウィンドウを常に最前面にする機能は実装していない |

### 横断検索

| 制約 | 状況 |
|------|------|
| Shell 横断検索は開いているタブのみが対象 | 閉じているファイル・最近使ったファイル履歴は検索対象に含まれない |

### ファイル形式

| 制約 | 状況 |
|------|------|
| 共通プロジェクトファイル形式は未導入 | `.nestsuite` は 1 タブ＝1 ファイルを維持したまま拡張子・保存経路を統一した wrapper であり、複数 Workspace を 1 ファイルへまとめる形式ではない。NoteNest / ChatNest / IdeaNest の保存内容（schema）自体は互いに独立している |
| NoteNest 保存スキーマは 1.4.2 で固定 | 形式変更予定なし |

### アーキテクチャ

| 制約 | 状況 |
|------|------|
| NoteNest は暫定的にタブごとに `MainViewModel` を持つ | 将来的に `NoteNestWorkspaceSessionViewModel` への軽量化を検討 |
